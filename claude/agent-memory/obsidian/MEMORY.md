## Note Format

Filename: `YYYYMMDDHHmm Title.md`

```markdown
## YYYYMMDDHHmm Title

{body content}

---
YYYYMMDDHHmm

#SubjectTag
#Meta--元数据/Source--来源/Claude-Code

```

Rules:
- H2 heading = `ID + space + Title` (same as filename without extension)
- Body: drafted content; may be empty for stub captures
- `---` separator line
- ID repeated on its own line
- One blank line
- Tags, one per line (subject tags first, meta tags last)
- File ends with a space on the last line, then a newline

## Tag System

### Naming convention
- `#English--中文/Subcategory--中文`
- Multi-word for autocomplete: `#WIP--Work-In-Progress--进行中`

### Meta tags (always at bottom, describe the note itself)

| Tag | Meaning |
|-----|---------|
| `#Meta--元数据/Source--来源/Claude-Code` | Generated by Claude Code |
| `#Meta--元数据/Status--状态/WIP--Work-In-Progress--进行中` | Unfinished note |
| `#Meta--元数据/Type--类型/MOC--Map-of-Content--目录` | Index/navigation page |
| `#Meta--元数据/Type--类型/Flashcards--记忆卡片` | Flashcard note |
| `#Meta--元数据/Type--类型/Collection--合集` | Collection note |

### Subject tags (common examples)

```
#Productivity--效率/Claude-Code
#Productivity--效率/Claude-Code/Memories--记忆管理
#Productivity--效率/Claude-Code/Skills--技能
#Productivity--效率/PKM--Personal-Knowledge-Management--个人知识管理
#Tech--技术/Obsidian
```

Subject tags live under their domain hierarchy. When in doubt, use a broad parent tag.
